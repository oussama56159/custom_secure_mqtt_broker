Dynamic Security Commands Cheat Sheet
=====================================

Because Mosquitto is already in the Windows PATH, use:
mosquitto_ctrl

Broker used in this project:
host: localhost
port: 8883 (TLS enabled)
admin user: admin
admin password: stored in secure-mqtt-broker/admin-password.txt

IMPORTANT: This broker requires BOTH certificate AND username/password authentication!

In all commands below, replace:
<ADMIN_PASSWORD> with the admin password (default: 906271)
<USERNAME> with the client/user name
<PASSWORD> with the client password
<ROLE_NAME> with the role name
<GROUP_NAME> with the group name
<TOPIC> with the topic, for example sensors/temperature

Base command format with TLS:
mosquitto_ctrl -h localhost -p 8883 --cafile "C:\custom_mqtt_broker\certificates\ca.crt" --cert "C:\custom_mqtt_broker\certificates\admin_client.crt" --key "C:\custom_mqtt_broker\certificates\admin_client.key" -u admin -P <ADMIN_PASSWORD> dynsec <COMMAND>

4. Publish Topic Permissions
----------------------------

To allow a user to publish on a topic:
1. create a role
2. add publishClientSend ACL to the role
3. assign the role to the user

Create publisher role:
mosquitto_ctrl -h localhost -p 8883 --cafile "C:\custom_mqtt_broker\certificates\ca.crt" --cert "C:\custom_mqtt_broker\certificates\admin_client.crt" --key "C:\custom_mqtt_broker\certificates\admin_client.key" -u admin -P <ADMIN_PASSWORD> dynsec createRole <ROLE_NAME>

mosquitto_ctrl -h localhost -p 8883 --cafile "C:\custom_mqtt_broker\certificates\ca.crt" --cert "C:\custom_mqtt_broker\certificates\admin_client.crt" --key "C:\custom_mqtt_broker\certificates\admin_client.key" -u admin -P <ADMIN_PASSWORD> dynsec addRoleACL <ROLE_NAME> publishClientSend <TOPIC> allow

8. Blocking / Denying Access
----------------------------

This project already has default ACL access set to deny, so a user with no matching allow rule cannot publish or subscribe.

To block a user quickly (prevents NEW connections):
mosquitto_ctrl -h localhost -p 8883 --cafile "C:\custom_mqtt_broker\certificates\ca.crt" --cert "C:\custom_mqtt_broker\certificates\admin_client.crt" --key "C:\custom_mqtt_broker\certificates\admin_client.key" -u admin -P <ADMIN_PASSWORD> dynsec disableClient <USERNAME>

To unblock:
mosquitto_ctrl -h localhost -p 8883 --cafile "C:\custom_mqtt_broker\certificates\ca.crt" --cert "C:\custom_mqtt_broker\certificates\admin_client.crt" --key "C:\custom_mqtt_broker\certificates\admin_client.key" -u admin -P <ADMIN_PASSWORD> dynsec enableClient <USERNAME>

IMPORTANT: Disabling a user only prevents NEW connections. Existing connections remain active until they disconnect.
To force immediate disconnection: restart the broker or disconnect the client.

To remove only one permission from a role:
mosquitto_ctrl -h localhost -p 8883 --cafile "C:\custom_mqtt_broker\certificates\ca.crt" --cert "C:\custom_mqtt_broker\certificates\admin_client.crt" --key "C:\custom_mqtt_broker\certificates\admin_client.key" -u admin -P <ADMIN_PASSWORD> dynsec removeRoleACL <ROLE_NAME> <ACL_TYPE> <TOPIC> allow

Example:
mosquitto_ctrl -h localhost -p 8883 --cafile "C:\custom_mqtt_broker\certificates\ca.crt" --cert "C:\custom_mqtt_broker\certificates\admin_client.crt" --key "C:\custom_mqtt_broker\certificates\admin_client.key" -u admin -P 906271 dynsec removeRoleACL temperature_publisher publishClientSend sensors/temperature allow


9. Useful ACL Types
-------------------

publishClientSend:
Allows a client to publish/send messages to a topic.
Use this for sensors.

subscribePattern:
Allows a client to subscribe to a topic filter, for example sensors/#.
Use this for dashboards.

publishClientReceive:
Allows a subscribed client to receive messages from a topic.
Use this with subscribePattern for dashboards.

unsubscribePattern:
Allows unsubscribe from a topic filter.

This broker requires DUAL AUTHENTICATION:
1. Valid client certificate (signed by CA)
2. Valid username and password from dynamic security

Do not give the admin role to normal sensors or dashboards.
Only admin should manage users, roles, groups, and ACLs.

Client passwords are stored in: secure-mqtt-broker/client-passwords.txt
Default password for all clients: 906271

After changing roles or ACLs, use getClient, getRole, and getGroup to verify the result.

When you disable a user:
- They cannot make NEW connections
- Existing connections remain active until they disconnect
- To force immediate disconnect: restart broker or client

Certificates are stored in: C:\custom_mqtt_broker\certificates\
Each client needs their own certificate (e.g., temp_sensor.crt, client.crt, etc.)
