For the first time, you create the admin when you initialize the Dynamic Security file.
Command:
& 'C:\Program Files\mosquitto\mosquitto_ctrl.exe' dynsec init 'C:\custom_secure_mqtt_broker\secure-mqtt-broker\dynamic-security.json' admin 906271

This creates:
username: admin
password: 906271
role: admin
NOTE!!
Roles are created inside Dynamic Security, and they are saved in:
C:\custom_secure_mqtt_broker\secure-mqtt-broker\dynamic-security.json

Example command that creates a role:
& 'C:\Program Files\mosquitto\mosquitto_ctrl.exe' -h localhost -p 1884 -u admin -P 906271 dynsec createRole my_role
Then you add permissions to that role:
& 'C:\Program Files\mosquitto\mosquitto_ctrl.exe' -h localhost -p 1884 -u admin -P 906271 dynsec addRoleACL my_role publishClientSend sensors/mytopic allow
Then you assign it to a user:
& 'C:\Program Files\mosquitto\mosquitto_ctrl.exe' -h localhost -p 1884 -u admin -P 906271 dynsec addClientRole s1 my_role
So the full flow is:
1. createRole
2. addRoleACL
3. addClientRole



A user is the account that connects to the broker.


So:
user = who connects
role = what they are allowed to do

Example:
User: s1
Role: s1_publisher
Permission inside role: publish to sensors/s1
Assign A Role To A User

Command:
& 'C:\Program Files\mosquitto\mosquitto_ctrl.exe' -h localhost -p 1884 -u admin -P 906271 dynsec addClientRole USERNAME ROLE_NAME
Example:
& 'C:\Program Files\mosquitto\mosquitto_ctrl.exe' -h localhost -p 1884 -u admin -P 906271 dynsec addClientRole s1 s1_publisher
Remove role:
& 'C:\Program Files\mosquitto\mosquitto_ctrl.exe' -h localhost -p 1884 -u admin -P 906271 dynsec removeClientRole s1 s1_publisher

What add_pub.py Does

It creates a new publisher user.
Inside the file, you set:

PUBLISHER_NAME = "s1"
PUBLISHER_PASSWORD = "906271"
ALLOWED_TOPIC = "sensors/s1"
MAKE_ADMIN = False
When you run:
python add_pub.py
It does this:

1. create user s1
2. set password 906271
3. create role s1_publisher
4. add permission to role:
   allow publish to sensors/s1
5. assign role s1_publisher to user s1
6. save password reminder in client-passwords.txt
If:
MAKE_ADMIN = True
it also gives s1 the admin role.

What add_sub.py Does

It creates a new subscriber user.
Inside the file, you set:
SUBSCRIBER_NAME = "dash1"
SUBSCRIBER_PASSWORD = "906271"
ALLOWED_TOPIC = "sensors/#" means it gets from evryone 


When you run:
python add_sub.py
It does this:
1. create user dash1
2. set password 906271
3. create role dash1_subscriber
4. add permission to role:
   allow subscribe to sensors/#
5. add permission to role:
   allow receive messages from sensors/#
6. assign role dash1_subscriber to user dash1
7. save password reminder in client-passwords.txt
